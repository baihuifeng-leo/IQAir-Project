# 竞品作战台 · Competitive Workbench

给电商类目日常作战用的两块画布：**品牌价格带沙盘** 和 **竞品对位表**。
所有改动即时写回服务器，多人同时在线协同编辑，刷新、换电脑、换浏览器都在。

---

运行时数据全部落在 `/var/lib/workbench`（跑 Docker 的话是容器里的 `/data`）：

```

users.json        用户表，PIN 用 scrypt 加盐存储

```

**代码目录和数据目录是分开的。** 更新代码、重装服务都不碰数据。

## 用户与登录

**用户名 + 6 位数字 PIN**，就这么简单。

第一次启动时，如果 `/data/users.json` 不存在，会按 compose 里的 `ADMIN_USER` / `ADMIN_PIN` 建一个管理员（默认 `admin` / `123456`）。登录后页面会提醒你去改掉默认 PIN。

从 `⋯ → 用户管理` 进去：

- **管理员**：添加成员、重置任何人的 PIN、删除成员。删不掉自己，也删不掉最后一个管理员。
- **普通成员**：只能改自己的 PIN。
- PIN 用 scrypt 加盐哈希存，服务端拿不到明文；忘了就让管理员重置。

改完 `ADMIN_PIN` 环境变量**不会**改已存在的用户 —— 那个变量只在第一次建表时用一次。之后一律在界面里改。

会话是 HttpOnly 的签名 Cookie，默认 30 天。密钥存在 `/data/.session-secret`，重启不会把人踢下线；想强制所有人重新登录，删掉这个文件再重启。

## 协同编辑

打开页面就自动接上实时通道，不用点任何按钮。

- **在线成员**：顶栏右侧的头像圈，谁在线一目了然。
- **别人改了，你立刻看到**：对方停手 0.7 秒，改动就推到你屏幕上，附一条「小李更新了竞品对位」的提示。
- **你正在打字时不会被打断**：如果远端改动到了、而你的光标正停在这个页面的某个输入框里，它会先排队，等你离开输入框再落地。顶栏会显示「小李 有新改动」。
- **两个人同时改，不会互相覆盖**。价格带沙盘和竞品对位是两份独立文档，各自有版本号。提交时版本对不上，服务端会拿三方合并（`merge.js`）把两边的改动合到一起：

  | 情况 | 结果 |
  |---|---|
  | 你改 A 品牌价格，他改 B 品牌名字 | 两个都保留 |
  | 你加了个产品，他也加了个产品 | 两个都在 |
  | 你删了个产品，他没动过它 | 删掉 |
  | 你删了个产品，他刚好改过它 | 保留他的版本（不静默丢失别人的工作） |
  | 你俩改了**同一个格子** | 后提交的赢，前一个人会看到「已经把两边的改动合到一起了」 |

  合并发生时页面会提示你一声，不会悄悄改你的东西。

> 撤销（`Ctrl+Z`）只回退**你自己**这一步，然后同步给所有人。它不是"全局时光机"，别指望用它撤掉别人的操作。

## 部署到 Ubuntu 虚拟机

在虚拟机里，把解压后的目录放到任意位置（比如 `~/workbench`），然后：

```bash
cd ~/workbench
sudo bash install.sh
```

脚本会装好 Node、建一个不能登录的系统用户 `workbench`、把代码放进 `/opt/workbench`、数据目录建在 `/var/lib/workbench`，最后注册成开机自启的 systemd 服务。跑完它会直接告诉你访问地址。

打开 `http://<虚拟机IP>:8090`，用 `admin` / `123456` 登录，**第一件事去「⋯ → 用户管理」改掉默认 PIN**。

### 日常操作

```bash
journalctl -u workbench -f          # 实时日志
sudo systemctl restart workbench    # 重启
sudo systemctl stop workbench       # 停
sudo systemctl status workbench     # 看状态
```

### 改端口 / 改初始管理员

```bash
sudo systemctl edit --full workbench
```

改 `Environment=PORT=8090` 这些行，存盘后：

```bash
sudo systemctl daemon-reload && sudo systemctl restart workbench
```

> `ADMIN_USER` / `ADMIN_PIN` 只在 `users.json` **还不存在**时用一次。已经建过表了，改这两个变量没有任何效果 —— 之后一律在界面里管用户。

### 更新代码

把新版目录传上去，再跑一次 `sudo bash install.sh` 就行。它只覆盖代码，`/var/lib/workbench` 里的数据和用户表原样保留。已经改过的 `workbench.service` 也不会被覆盖。

### 防火墙

虚拟机开了 ufw 的话：

```bash
sudo ufw allow 8090/tcp
```

如果 Lucky 跑在别的机器上，更稳的是只放行它：

```bash
sudo ufw allow from <Lucky所在机器IP> to any port 8090 proto tcp
```

这样公网入口就只剩 Lucky 一个。

## 配 Lucky 反向代理

在 Lucky 的「Web 服务 → 反向代理」里加一条规则：

- **后端地址**：`http://<虚拟机IP>:8090`
- **开启 HTTPS**：建议开。Lucky 能自动签证书，不开的话 PIN 是明文过公网的。
- **不要开响应缓冲**：协同用的是 SSE 长连接。服务端已经带了 `X-Accel-Buffering: no`，Lucky 默认也不缓冲；万一实时推送不动了，先查这里。
- **传递 `X-Forwarded-Proto`**：Lucky 默认会加。服务靠它决定要不要给 Cookie 打 `Secure` 标记。

> 顺带一提：`ports` 那行把 8090 开在了宿主机上。如果 Lucky 和这个容器在同一台机器、同一个 Docker 网络里，可以把 `ports` 整段删掉，让它只在 Docker 内网可达，公网就只剩 Lucky 一个入口。

## 功能一：品牌价格带沙盘

纵轴价格带、横轴品牌，格子里是产品卡。

- **改文字** — 开启编辑后，点产品名或价格，输入即存。
- **图例与分类支持富文本** — 分类名可以 **加粗**、*斜体*、下划线。选中文字点悬浮的 B / I / U，或 `Ctrl+B` / `Ctrl+I` / `Ctrl+U`。粘贴会强制转成纯文本。
- **搬产品** — 拖动产品卡，跨品牌、跨价格带随便扔；扔到另一张卡上会插在它前后。
- **分类配色** — 卡片右上角的**圆点**选分类；左侧面板改颜色和名称，全盘和图例同步变。
- **`I` / `U`** — 斜体、下划线，对应你原来标记"重点款"的习惯。
- **框选批量改** — 在空白处按住鼠标拖出选框，一次选中多个产品；<kbd>Ctrl</kbd> 点击加选或取消。选中后可以**整批拖到别的格子**，或用底部工具条批量改分类、切斜体/下划线、删除。<kbd>Esc</kbd> 取消选择，<kbd>Delete</kbd> 删除选中。
- **加结构** — 左侧「新增品牌列 / 新增价格带」，格子里悬停会浮出「+ 添加产品」。
- **删除** — 品牌名、价格带、产品卡上悬停出现的 `×`。删品牌或价格带会连带里面的产品，有二次确认。
- 左下角实时算出产品数、品牌数、价格带数和均价。

## 功能二：竞品对位

对着你那页 PPT 一比一还原，但每一格都能改。

- **品牌列** — 每个品牌可以传三张图：Logo、**1:1 产品图**、**3:4 素材**。
- **原图不动** — 上传走裸二进制直传，服务端字节原样落盘，**不经 canvas 重编码、不转 base64**。之前那套 `toDataURL(0.92)` 必然重编码，字再也清晰不了。单张上限 40MB。
- **点图看原图** — 点击图片弹出 Lightbox，可切「适应窗口 / 100% 原图 / 新标签打开」，右下角显示原图像素尺寸。上传动作挪到了图片下方的独立入口（`↑ 上传` / `⟳ 更换` / `移除`），只在编辑模式下出现，不会再手滑覆盖素材。
- 悬停仍有小预览，快速扫一眼用；要看清字就点开。
- **一格多值** — 一个单元格可以放多行数值，每行有**大字**和**小注释**（对应 PPT 里的 `1000` / `CADR颗粒物`）。悬停出现「+ 数值」加行。
- **红蓝高亮** — 点数值左边的**小圆点**循环切换：中性 → 蓝色（我方优势）→ 红色（我方劣势）。
- **我方品牌** — 左侧选中后，那一列的表头会被红框圈住、贴上「我方」角标，下面每个单元格也带红色边线。
- **加结构** — 「新增品牌列」「新增参数分组」，每个分组卡上悬停出现「+ 参数行」。

### 中英文切换

左侧「语言」切到 English，整张表切换到英文内容。**中英文各存一份**，互不覆盖。

- 改了中文，对应的英文会被自动标成**待更新**（琥珀色底）。
- 英文还空着的地方，输入框会高亮并提示「待更新英文信息」。
- 手动改过英文之后，待更新标记自动消失。
- 侧栏实时显示「N 处缺英文，N 处待更新」，全部跟上时变成绿色。

> 服务端不做机器翻译（那需要联网调外部 API，而这台机器是内网自用的）。所以是**标记提示**而不是自动翻译 —— 改了中文你会立刻看到哪些英文需要跟着改，不会有"以为同步了其实没同步"的假象。
>
> 老数据升级时，英文字段会按 PPT 原文自动补齐一次；补不上的留空，界面会标出来。

## 编辑模式

**页面默认只读。** 右上角点「开启编辑」才能改东西，防止日常浏览时手滑。

只读态下：拖拽、框选、删除按钮、上传入口、高亮圆点全部关闭；远端同事的改动照常实时推送过来。状态记在浏览器里，下次打开还是上次那个模式。

## 功能三：评论风向标

### 它统计什么

**不是「好评数 vs 差评数」。** 你的 Excel 里没有评分列，而淘宝晒单评论天然一边倒——实测 6,332 条里，算法能识别的负向分句只有 2%，且人工核对下来约四成是误判（"家里电梯坏了"说的是电梯，"不会很占地方"其实是夸）。做成好评差评柱状图，七个品牌会清一色 97% 好评，好看但没用。

**统计的是「维度 × 极性」。** 一条"静音很好，就是滤芯太贵"同时贡献 `噪音:正向` + `滤芯成本:负向`。看板上是一张 品牌 × 9 个维度 的热力图，颜色越红代表这个维度被吐槽得越多。样本量小的格子会自动变淡——3 条提及里 1 条负面，说明不了什么。

九个维度：净化效果 · 异味处理 · 噪音 · 滤芯成本 · 质量做工 · 外观设计 · 体积重量 · 操作智能 · 服务物流。

> 想要真正的好评/差评统计，让运营导出时勾上**评分列**，或直接导「中评+差评」标签页。有星级就是一次分组统计，根本不需要 NLP。

### 关键词与溯源

好评词云、差评词云，字号按出现次数。**鼠标停在任意关键词上，浮层里显示它在原始评论中的上下文原文**，按「有用」数排序，可以滚动看多条。点侧栏的品牌名可以只看这个品牌的词。

### 导入 Excel

侧栏填品牌名，选一份 `.xlsx`（或直接拖进去）。

- **自动去重**：指纹 = `品牌 + SKU + 旺旺号 + 日期 + 类型 + 正文摘要`，只追加库里没有的。
- **不能只按正文哈希去重**——实测那样会误删 44 条不同买家写出的相同真实短评（"收到后开机用了一天，完全没闻到异味"出现 9 次，来自 8 个买家）。
- 「追评」是一条独立评论，有自己的时间和图，拆成单独一行入库。
- 淘宝模板评论（"该用户觉得商品非常好，给出5星好评"占 24%）会被标记并排除出统计，但不会被删掉——删了你就不知道有多少人懒得写评价。
- 导入完显示报告：新增 / 跳过已存在 / **这份文件自己有多少行重复**。

管理员可以删除某个品牌的全部评论。所有导入删除都进变更日志。

## 功能四：竞品3D预览

价格、颗粒物CADR、甲醛CADR 三个轴摆成一张可交互的 3D 散点图（ECharts GL），拖动旋转视角、滚轮缩放，气泡越大代表性价比（两项 CADR 之和 / 价格）越高，颜色按品牌区分。鼠标停在气泡上看详细参数，点一下直接跳转商品链接。

### 导入 Excel

侧栏「导入 / 更新 Excel」，表头必须包含：**品牌、型号、商品ID、价格、颗粒物CADR、甲醛CADR、商品链接**。

- **每次导入是整份覆盖**，不是逐条追加——这份表格代表"当前最新价目/参数"，旧行理应被新表格取代，不会有删不掉的僵尸数据。评论库是历史记录所以增量合并，这里是现状快照所以整份替换，两种模型刻意不同。
- 品牌颜色单独持久化，不随每次导入清空：同一个品牌换了几轮表格颜色也不会跳来跳去。
- 侧栏点品牌名可以单独隐藏它，方便只对比剩下的几家；「全部显示」一键恢复。

## 备份与恢复

**每天本地时间 00:00 自动备份一次，滚动保留最近 30 份。** 服务启动时会在日志里打印下次备份时间。

管理员从 `⋯ → 备份与恢复` 进去，可以：

- 看到所有备份，标着是「每日自动」「手动」还是「恢复前存档」
- 点「立刻备份一次」手动存一份
- 点任意一份的「恢复到这里」把**所有人**的数据回滚过去

恢复是有后路的：动手之前，服务端会先把当前状态另存成一份 `before-restore` 备份。恢复完成后所有在线的人会立刻看到新内容。

非管理员看不到这两个菜单项，接口层面也会直接回 403。

### 整机搬迁

```bash
# 备份（服务可以不停）
sudo tar czf ~/workbench-backup-$(date +%F).tgz -C /var/lib/workbench .

# 还原
sudo systemctl stop workbench
sudo tar xzf ~/workbench-backup-2026-07-09.tgz -C /var/lib/workbench
sudo chown -R workbench:workbench /var/lib/workbench
sudo systemctl start workbench
```

界面里的「导出 JSON 备份」只导两块画布的内容，不含图片和用户表；要整体搬家用上面的 tar。

## 变更日志

`⋯ → 变更日志`，**仅管理员可见**。记录每一次登录、编辑、加人删人、上传、备份和恢复。

编辑记录不是"某某改了沙盘"这种废话，而是具体到字段：

```
li · doc.save [matrix]
    新增 品牌「李加的品牌」
    产品 › GCX XE › 价格：「¥28,188」→「¥88,888」

zhang · doc.save [compare]
    标题：「Competitive Challenge」→「新标题」
    分组 › 核心性能 › 参数行 › CADR 参数 › 单元格 › 第 1 行 › 数值：「1000」→「1200」
```

面板顶部可以按用户名或内容过滤。日志写在 `audit.log`，超过 8MB 自动轮转成 `audit.log.1`。

## 几点说明

- 会话过期时页面会提示 3 秒再跳登录页，未同步的改动不会静默丢失。
- 图片在浏览器端就压到最长边 720px 再上传，只收 PNG / JPG / WebP。
- 页面不请求任何外部 CDN 或字体，断网的内网环境也能正常打开。
- 「导出 JSON」「从备份恢复」「恢复出厂数据」影响的是**所有人**看到的那份数据，不是你本地的副本。
- **手机端**：侧栏变抽屉，触控目标放大。但对位表是 5 列 × 11 行的二维表，**手机上不可能不横滚**——硬折成一维卡片流会丢掉"横向对比"这个核心价值。所以保留横滚 + `scroll-snap`，每次滑动干净地停在一个品牌列上。
- 产品图现在是**原图**，不压缩。一张 8MB 的图就是 8MB，图多了记得留意 `uploads/` 的体积。
- 评论库单独放 `reviews/reviews.jsonl`，不进 `db.json`。7 个品牌 6,332 条约 4.2MB，聚合一次 190ms。超过 20 万条再考虑换 SQLite。

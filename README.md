# 电商工作台 · E-commerce Workbench

给电商类目日常作战用的工作台：价格带沙盘、竞品对位、评论风向标、竞品3D预览、报告管理。
所有改动即时写回服务器，多人同时在线协同编辑，刷新、换电脑、换浏览器都在。

## 用户与登录

**用户名 + 6 位数字 PIN**，就这么简单。

第一次启动时，如果 `/data/users.json` 不存在，会按 compose 里的 `ADMIN_USER` / `ADMIN_PIN` 建一个管理员（默认 `admin` / `123456`）。登录后页面会提醒你去改掉默认 PIN。

从 `⋯ → 用户管理` 进去：

- **管理员**：添加成员、重置任何人的 PIN、删除成员。删不掉自己，也删不掉最后一个管理员。
- **普通成员**：只能改自己的 PIN。
- PIN 用 scrypt 加盐哈希存，服务端拿不到明文；忘了就让管理员重置。

改完 `ADMIN_PIN` 环境变量**不会**改已存在的用户 —— 那个变量只在第一次建表时用一次。之后一律在界面里改。

会话是 HttpOnly 的签名 Cookie，默认 30 天。密钥存在 `/data/.session-secret`，重启不会把人踢下线；想强制所有人重新登录，删掉这个文件再重启。

## 协同编辑

打开页面就自动接上实时通道，不用点任何按钮。

- **在线成员**：顶栏右侧的头像圈，谁在线一目了然。
- **别人改了，你立刻看到**：对方停手 0.7 秒，改动就推到你屏幕上，附一条「小李更新了竞品对位」的提示。
- **你正在打字时不会被打断**：如果远端改动到了、而你的光标正停在这个页面的某个输入框里，它会先排队，等你离开输入框再落地。顶栏会显示「小李 有新改动」。
- **两个人同时改，不会互相覆盖**。价格带沙盘和竞品对位是两份独立文档，各自有版本号。提交时版本对不上，服务端会拿三方合并（`merge.js`）把两边的改动合到一起：

  | 情况 | 结果 |
  |---|---|
  | 你改 A 品牌价格，他改 B 品牌名字 | 两个都保留 |
  | 你加了个产品，他也加了个产品 | 两个都在 |
  | 你删了个产品，他没动过它 | 删掉 |
  | 你删了个产品，他刚好改过它 | 保留他的版本（不静默丢失别人的工作） |
  | 你俩改了**同一个格子** | 后提交的赢，前一个人会看到「已经把两边的改动合到一起了」 |

  合并发生时页面会提示你一声，不会悄悄改你的东西。

> 撤销（`Ctrl+Z`）只回退**你自己**这一步，然后同步给所有人。它不是"全局时光机"，别指望用它撤掉别人的操作。

## 部署到 Ubuntu 虚拟机

在虚拟机里，把解压后的目录放到任意位置（比如 `~/workbench`），然后：

```bash
cd ~/workbench
sudo bash install.sh
```

脚本会装好 Node、建一个不能登录的系统用户 `workbench`、把代码放进 `/opt/workbench`、数据目录建在 `/var/lib/workbench`，最后注册成开机自启的 systemd 服务。跑完它会直接告诉你访问地址。

打开 `http://<虚拟机IP>:8090`，用 `admin` / `123456` 登录，**第一件事去「⋯ → 用户管理」改掉默认 PIN**。

### 日常操作

```bash
journalctl -u workbench -f          # 实时日志
sudo systemctl restart workbench    # 重启
sudo systemctl stop workbench       # 停
sudo systemctl status workbench     # 看状态
```

### 改端口 / 改初始管理员

```bash
sudo systemctl edit --full workbench
```

改 `Environment=PORT=8090` 这些行，存盘后：

```bash
sudo systemctl daemon-reload && sudo systemctl restart workbench
```

> `ADMIN_USER` / `ADMIN_PIN` 只在 `users.json` **还不存在**时用一次。已经建过表了，改这两个变量没有任何效果 —— 之后一律在界面里管用户。

### 更新代码

把新版目录传上去，再跑一次 `sudo bash install.sh` 就行。它只覆盖代码，`/var/lib/workbench` 里的数据和用户表原样保留。已经改过的 `workbench.service` 也不会被覆盖。

### 防火墙

虚拟机开了 ufw 的话：

```bash
sudo ufw allow 8090/tcp
```

如果 Lucky 跑在别的机器上，更稳的是只放行它：

```bash
sudo ufw allow from <Lucky所在机器IP> to any port 8090 proto tcp
```

这样公网入口就只剩 Lucky 一个。

## 配 Lucky 反向代理

在 Lucky 的「Web 服务 → 反向代理」里加一条规则：

- **后端地址**：`http://<虚拟机IP>:8090`
- **开启 HTTPS**：建议开。Lucky 能自动签证书，不开的话 PIN 是明文过公网的。
- **不要开响应缓冲**：协同用的是 SSE 长连接。服务端已经带了 `X-Accel-Buffering: no`，Lucky 默认也不缓冲；万一实时推送不动了，先查这里。
- **传递 `X-Forwarded-Proto`**：Lucky 默认会加。服务靠它决定要不要给 Cookie 打 `Secure` 标记。

> 顺带一提：`ports` 那行把 8090 开在了宿主机上。如果 Lucky 和这个容器在同一台机器、同一个 Docker 网络里，可以把 `ports` 整段删掉，让它只在 Docker 内网可达，公网就只剩 Lucky 一个入口。

## 功能一：品牌价格带沙盘

纵轴价格带、横轴品牌，格子里是产品卡。

- **改文字** — 开启编辑后，点产品名或价格，输入即存。
- **图例与分类支持富文本** — 分类名可以 **加粗**、*斜体*、下划线。选中文字点悬浮的 B / I / U，或 `Ctrl+B` / `Ctrl+I` / `Ctrl+U`。粘贴会强制转成纯文本。
- **搬产品** — 拖动产品卡，跨品牌、跨价格带随便扔；扔到另一张卡上会插在它前后。
- **分类配色** — 卡片右上角的**圆点**选分类；左侧面板改颜色和名称，全盘和图例同步变。**打了分类的卡片整卡背景填色**（不再只是左边一条细色条），文字颜色（黑/白）跟着背景亮度自动切换，保证任何颜色都读得清；没打分类的产品仍然是原来的白卡片——分类色块的意义是让"有标记的产品"跳出来，如果所有产品都上色，矩阵反而会变成一片糊在一起看不出重点。
- **"新品"分类自动带 New 角标** — 分类名字里带"新品"或"New"字样（不区分大小写，比如你们现在用的"Y26 New Product Launch 新品"），这个分类下的产品卡左上角会自动挂一个醒目的粉红色 New 标签，不用再逐个产品单独标记。
- **`I` / `U`** — 斜体、下划线，对应你原来标记"重点款"的习惯。
- **框选批量改** — 在空白处按住鼠标拖出选框，一次选中多个产品；<kbd>Ctrl</kbd> 点击加选或取消。选中后可以**整批拖到别的格子**，或用底部工具条批量改分类、切斜体/下划线、删除。<kbd>Esc</kbd> 取消选择，<kbd>Delete</kbd> 删除选中。
- **加结构** — 左侧「新增品牌列 / 新增价格带」，格子里悬停会浮出「+ 添加产品」。
- **删除** — 品牌名、价格带、产品卡上悬停出现的 `×`。删品牌或价格带会连带里面的产品，有二次确认。
- 左下角实时算出产品数、品牌数、价格带数和均价。

### 导出 PNG（16:9，给 PPT 用）

标题右上角「⭳ 导出 PNG（16:9）」，一键把整张矩阵（含标题、图例）导出成一张固定 16:9 比例的图片，直接拖进 PPT 就能用，不用再用截图工具手动拼、调比例。

- **自动收窄列宽**：品牌一多，矩阵天然会很宽——导出前会自动试着把列宽收窄，让内容本身的宽高比更接近 16:9，减少画面里的空白；列宽收窄是有下限的（低于这个宽度产品名会挤得没法看），到下限还没凑够比例就不再收窄了。
- **等比缩放兜底**：收窄完不一定刚好是 16:9，剩下的差值用整体等比缩放 + 居中留白兜底，保证输出的图片**一定是严格的 16:9**，不会出现内容被拉伸变形的情况。
- 导出内容是当前矩阵的静态快照，编辑态才有意义的拖拽手柄、增删按钮这些不会出现在图里。

## 功能二：竞品对位

对着你那页 PPT 一比一还原，但每一格都能改。

- **品牌列** — 每个品牌可以传三张图：Logo、**1:1 产品图**、**3:4 素材**。
- **原图不动** — 上传走裸二进制直传，服务端字节原样落盘，**不经 canvas 重编码、不转 base64**。之前那套 `toDataURL(0.92)` 必然重编码，字再也清晰不了。单张上限 40MB。
- **点图看原图** — 点击图片弹出 Lightbox，可切「适应窗口 / 100% 原图 / 新标签打开」，右下角显示原图像素尺寸。上传动作挪到了图片下方的独立入口（`↑ 上传` / `⟳ 更换` / `移除`），只在编辑模式下出现，不会再手滑覆盖素材。
- 悬停仍有小预览，快速扫一眼用；要看清字就点开。
- **一格多值** — 一个单元格可以放多行数值，每行有**大字**和**小注释**（对应 PPT 里的 `1000` / `CADR颗粒物`）。悬停出现「+ 数值」加行。
- **红蓝高亮** — 点数值左边的**小圆点**循环切换：中性 → 蓝色（我方优势）→ 红色（我方劣势）。
- **我方品牌** — 左侧选中后，那一列的表头会被红框圈住、贴上「我方」角标，下面每个单元格也带红色边线。
- **加结构** — 「新增品牌列」「新增参数分组」，每个分组卡上悬停出现「+ 参数行」。

### 中英文切换

左侧「语言」切到 English，整张表切换到英文内容。**中英文各存一份**，互不覆盖。

- 改了中文，对应的英文会被自动标成**待更新**（琥珀色底）。
- 英文还空着的地方，输入框会高亮并提示「待更新英文信息」。
- 手动改过英文之后，待更新标记自动消失。
- 侧栏实时显示「N 处缺英文，N 处待更新」，全部跟上时变成绿色。

> 服务端不做机器翻译（那需要联网调外部 API，而这台机器是内网自用的）。所以是**标记提示**而不是自动翻译 —— 改了中文你会立刻看到哪些英文需要跟着改，不会有"以为同步了其实没同步"的假象。
>
> 老数据升级时，英文字段会按 PPT 原文自动补齐一次；补不上的留空，界面会标出来。

## 编辑模式

**页面默认只读。** 右上角点「开启编辑」才能改东西，防止日常浏览时手滑。

只读态下：拖拽、框选、删除按钮、上传入口、高亮圆点全部关闭；远端同事的改动照常实时推送过来。状态记在浏览器里，下次打开还是上次那个模式。

这个开关只对**价格带沙盘、竞品对位**这两个协同编辑文档有意义。评论风向标、竞品3D预览、报告管理三个功能本身没有"编辑态"这回事（各自走自己的导入/填表流程），切到这几个标签页时右上角不会显示这个按钮，省得点了没反应。

同理，「撤销 / 重做」也只对这两个协同编辑文档有意义（撤销栈只记录矩阵/对位的改动）——切到评论风向标、竞品3D预览、报告管理这几个标签页时，右上角也不会显示撤销/重做按钮。

## 功能三：评论风向标

### 它统计什么

**不是「好评数 vs 差评数」。** 你的 Excel 里没有评分列，而淘宝晒单评论天然一边倒——实测 6,332 条里，算法能识别的负向分句只有 2%，且人工核对下来约四成是误判（"家里电梯坏了"说的是电梯，"不会很占地方"其实是夸，"才明白之前的净化器都是智商税"是在贬旧褒新、夸的是当前买的这台）。识别规则会持续跟着这类真实误判案例迭代，做成好评差评柱状图，七个品牌会清一色 97% 好评，好看但没用。

**统计的是「维度 × 极性」。** 一条"静音很好，就是滤芯太贵"同时贡献 `噪音:正向` + `滤芯成本:负向`。看板上是一张 品牌 × 9 个维度 的热力图。

**颜色深浅按差评的绝对句数排名，不是差评占比。** 早期版本按"差评句数 / 总句数"的比例决定颜色，会出现"3条提及里1条负面（差评率100%）"比"200条里50条负面（差评率25%）"颜色还深的反常情况——评论基数小的格子因为一条差评就冲到最深红，反而比真正被大量吐槽的格子更显眼。现在改成：格子里的大字是差评句数，这个数字在全场（所有品牌 × 所有维度）里排第几决定颜色深浅，差评数越多颜色越深；好评句数、差评占比仍然显示在格子的小字里，作为参考不参与颜色计算。

九个维度：净化效果 · 异味处理 · 噪音 · 滤芯成本 · 质量做工 · 外观设计 · 体积重量 · 操作智能 · 服务物流。

**有差评的格子可以直接悬浮看原句。** 鼠标停在任意一个差评数大于 0 的格子上，会弹出这个品牌在这个维度下的差评原文列表（按「有用」数排序），跟关键词云的原文溯源是同一套浮层——不用先点品牌筛选再去关键词云里找，热力图上直接看。

> 想要真正的好评/差评统计，让运营导出时勾上**评分列**，或直接导「中评+差评」标签页。有星级就是一次分组统计，根本不需要 NLP。

### 关键词与溯源

好评词云、差评词云，字号按出现次数。**鼠标停在任意关键词上，浮层里显示它在原始评论中的上下文原文**，按「有用」数排序，可以滚动看多条。点侧栏的品牌名可以只看这个品牌的词。

### 导入 Excel

侧栏填品牌名，选一份 `.xlsx`（或直接拖进去）。

- **自动去重**：指纹 = `品牌 + SKU + 旺旺号 + 日期 + 类型 + 正文摘要`，只追加库里没有的。
- **不能只按正文哈希去重**——实测那样会误删 44 条不同买家写出的相同真实短评（"收到后开机用了一天，完全没闻到异味"出现 9 次，来自 8 个买家）。
- 「追评」是一条独立评论，有自己的时间和图，拆成单独一行入库。
- 淘宝模板评论（"该用户觉得商品非常好，给出5星好评"占 24%）会被标记并排除出统计，但不会被删掉——删了你就不知道有多少人懒得写评价。
- 导入完显示报告：新增 / 跳过已存在 / **这份文件自己有多少行重复**。

管理员可以删除某个品牌的全部评论。所有导入删除都进变更日志。

### 本品分析

前面说的「品牌 × 维度」是**竞品对比**——多个品牌摆在一起看谁在哪个维度更弱。画布右上角切到**本品分析**，是另一个独立场景：只针对 IQAir 自己的某一款产品，做深度的评价清洗和分析，不跟竞品混在一起看。

- **先建/选产品，再导入**：左侧「选择 / 新建产品」填个型号名（比如「GC-Multi」），选中它之后再导入这个型号专属的评论 `.xlsx`。产品之间的数据完全分开统计。
- **产品数据会自动汇总进 IQAir 这个品牌**：本品分析的记录同时也算作「IQAir」这个品牌的评论——如果哪天也想在竞品对比的热力图里看到 IQAir 跟其他品牌放在一起比，数据已经在那儿了，不用重新导一遍。
- 统计卡片、维度总览条形图、关键词云、差评原句悬浮溯源——跟竞品对比是同一套看板组件，只是数据源换成了这一款产品自己的评论，不需要重新学一套看法。
- 管理员可以删除某个产品的全部评论（连带产品本身一起删）。

## 功能四：竞品3D预览

**价格摆在竖直方向（纵轴）**，水平面两个方向分别是颗粒物CADR、甲醛CADR，摆成一张可交互的 3D 散点图（ECharts GL），滚轮缩放，颜色按品牌区分，气泡上方常显**品牌名 + 型号**两行文字。鼠标停在气泡上看详细参数，点一下直接跳转商品链接。

> 技术细节：ECharts-GL 的 3D 坐标系里，视觉上"竖起来"的那根轴其实是 `zAxis3D`，`xAxis3D`/`yAxis3D` 才是水平面的两个方向——容易被技术轴名带偏，这里特意把「价格」配置给 `zAxis3D`。

### 气泡大小可切换口径

三根轴已经占满了空间，销售表现类的数据不硬塞成第四根轴，而是改用**气泡大小**来承载——画布右上角可以在三种口径之间切换：

- **性价比**（默认）：两项 CADR 之和 / 价格，越高气泡越大。
- **5-6月销售额**：气泡越大代表这段时间卖得越多钱。
- **5-6月销量**：气泡越大代表这段时间卖出的件数越多。

切到销售额/销量模式时，如果当前数据没有对应列（比如导入的是老表格），气泡会全部显示为最小并弹出提示，不会误导成"大家都卖得一样差"。

气泡上方的品牌名 + 型号标签是常显的：3D 视角旋转时相近的点可能会互相遮挡、字压字，这是三维空间文字标签的固有局限——鼠标停留看 tooltip 里的完整参数更准确。

### 自动旋转开关

画布右上角「⟲ 自动旋转」是个独立开关：**开**的时候画布自动缓慢转动，方便无人值守时展示；**关**了以后完全交给手动拖拽控制视角，不会转到一半被自动旋转打断。滚轮缩放两种模式下都能用。

### 全屏模式

画布右上角「⛶ 全屏」只对图表这个区域发起全屏，不是整个浏览器窗口——侧栏和顶栏天然被排除在外（这是浏览器全屏 API 本身的语义，不用额外写代码去隐藏它们）。进入全屏后：

- **气泡和标签文字自动放大**：光把画布撑大不够，气泡基础大小和字号也会跟着放大一档，不然占屏比例反而变小，越看越费劲。
- 统计卡片区域让位给图表本身，把尽可能多的空间留给三维场景。
- 自动旋转开关照常显示在全屏画面里，不用退出全屏就能切换。
- 按 <kbd>Esc</kbd> 或再点一次按钮退出全屏，图表状态自动同步复原。

### 导入 Excel

侧栏「导入 / 更新 Excel」，表头必须包含：**品牌、型号、商品ID、价格、颗粒物CADR、甲醛CADR、商品链接**；可以再加两列可选的 **「5-6月销售额」「5-6月销量」**，用来支持上面的气泡口径切换——没有这两列也能正常导入，只是气泡在这两个口径下会显示为 0。

- **每次导入是整份覆盖**，不是逐条追加——这份表格代表"当前最新价目/参数"，旧行理应被新表格取代，不会有删不掉的僵尸数据。评论库是历史记录所以增量合并，这里是现状快照所以整份替换，两种模型刻意不同。
- 品牌颜色单独持久化，不随每次导入清空：同一个品牌换了几轮表格颜色也不会跳来跳去。
- 侧栏点品牌名可以单独隐藏它，方便只对比剩下的几家；「全部显示」一键恢复。

## 功能五：报告管理（第一期：个人报告）

原本是每周手工做一份 PPT 汇报，前两页固定模板、后面自定义图文。这里已经把**第一页、第二页**做成图表（对应右上角「第 1 页 · 生意参谋」「第 2 页 · 微盟数据」两个页签，分开展示不挤在一屏），支持全屏放映，后面再做自定义图文页和查看/编辑切换。

「个人报告」是每个人自己的数据，不是团队共享文档——你导入的表格只有你自己能看到。「公共报告」是团队共享看板，还在开发中，入口先留着。

### 导入一份 Excel，两种口径都有

侧栏一个入口，导入 `.xlsx`，表头需含：**统计时段、浏览量（店铺）、访客数（店铺）、浏览量（首页）、访客数（首页）、点击次数、点击人数、点击率、跳失率、平均停留时长(秒)、引导下单金额、引导下单买家数、引导下单转化率、引导支付金额、引导支付买家数、引导支付转化率、引导商品详情次数、引导商品详情人数、引导加购人数、引导加购件数**。

真正的数据不一定要在第一个 sheet——读取器会把每个 sheet 都扫一遍，找表头对得上的那个（导出文件前面常有透视表/对比表之类的辅助 sheet）。

- **按日期增量合并**：已经有的日期用新数据覆盖，其余日期不受影响。所以每周传同一份表格、时间窗口有重叠也没关系，只会追加新内容，不用自己去重。
- **目前只支持 `.xlsx`**。生意参谋原生导出常是老版 `.xls` 格式，本项目的零依赖读取器解析不了那个二进制格式——先用 Excel/WPS 把文件另存为 `.xlsx` 再上传。
- 百分比列（点击率、跳失率……）不管是文本 `"22.00%"` 还是 Excel 百分比格式存的小数 `0.22`，读取器都能识别正确换算——后者是靠读 `styles.xml` 里的数字格式，不是瞎猜小数点位置。

### 访客 / 浏览趋势 · 店铺整体

用「浏览量（店铺）/访客数（店铺）」两列画折线图。右上角是时间范围，不是手动选粒度——默认「本年」按周聚合，「上周/本周」这种窄范围自动切回按日，自定义范围按跨度自适应（≤31 天=日，≤370 天=周，更长=月）。范围决定粒度，少一层"该选哪个粒度"的心智负担。目前只统计 IQAir天猫旗舰店。图表下方另有「上周 vs 上上周」的浏览量/访客数对比——特意不用"本周"，因为本周通常还没过完，跟一个不完整的周比涨跌幅会失真。

### 生意参谋指标 · 首页 · 上周 vs 本周

对应原 PPT 第一页那 7 组对比图（浏览量/访客数、点击率/跳失率、停留时长、详情页、加购、点击、支付金额），用的是「（首页）」那两列加其余指标，范围比店铺整体窄。自动取最新 14 天的数据，按日期切成两组各 7 天当「上周」「本周」；计数类指标（浏览量、点击次数……）取总和，比率类指标（点击率、停留时长……）取平均值，涨跌幅自动算。

### 微盟数据 · 公众号 / 小程序 / APP（第 2 页）

对应原 PPT 第二页，右上角页签切过去单独一页展示，不跟第一页的图表挤在一起。微盟（公众号/小程序/APP 三端商城）后台没有导出接口，这部分不走 Excel 导入，改成侧栏「记录 / 编辑某周数据」手动填：8 个指标（浏览量、访客数、访问次数、平均访问深度、点击人数、点击次数、人均停留时长、跳出率）+ 浏览量/访客数各自的三端拆分。

- **可以补填任意一周**：表单是"选周"控件（不是选某一天），不会有选到周几都行、算出来的周一却对不上的问题；忘了填的老数据随时能补上。再次填同一周会带出已存的旧值（编辑而不是从空白开始），保存后是覆盖更新，不会变成重复记录。
- **环比按自然周算，不是按填表顺序**：永远比较"这一周的前一个自然周"，所以先补哪周、后补哪周都不影响涨跌幅对不对。
- 页面上方有个周份下拉框，可以回看任意一周的历史记录，配一个「编辑这一周」按钮直接改。
- **自动生成一句话点评**：挑浏览量/访客数六个渠道拆分里波动最大的那一项，拼成"本周全渠道流量及互动{普遍上涨/下滑}，其中{渠道}{涨/跌}幅达 X%，是主要{拉高/拉低}项"，呼应原 PPT 的叙述风格。

### 放映模式

右上角「▶ 放映模式」进全屏、自动隐藏顶栏和侧栏这些跟内容无关的东西，把图表和数字放大到适合投屏的尺寸；用左右方向键在两页之间切换，按 <kbd>Esc</kbd> 或点右上角「✕ 退出放映」退出，会议室里直接投屏用，不用切到打印视图。

## 备份与恢复

**每天本地时间 00:00 自动备份一次，滚动保留最近 30 份。** 服务启动时会在日志里打印下次备份时间。

管理员从 `⋯ → 备份与恢复` 进去，可以：

- 看到所有备份，标着是「每日自动」「手动」还是「恢复前存档」
- 点「立刻备份一次」手动存一份
- 点任意一份的「恢复到这里」把**所有人**的数据回滚过去

恢复是有后路的：动手之前，服务端会先把当前状态另存成一份 `before-restore` 备份。恢复完成后所有在线的人会立刻看到新内容。

非管理员看不到这两个菜单项，接口层面也会直接回 403。

### 整机搬迁

```bash
# 备份（服务可以不停）
sudo tar czf ~/workbench-backup-$(date +%F).tgz -C /var/lib/workbench .

# 还原
sudo systemctl stop workbench
sudo tar xzf ~/workbench-backup-2026-07-09.tgz -C /var/lib/workbench
sudo chown -R workbench:workbench /var/lib/workbench
sudo systemctl start workbench
```

界面里的「导出 JSON 备份」只导两块画布的内容，不含图片和用户表；要整体搬家用上面的 tar。

## 变更日志

`⋯ → 变更日志`，**仅管理员可见**。记录每一次登录、编辑、加人删人、上传、备份和恢复。

li · doc.save [matrix]
    新增 品牌「李加的品牌」

面板顶部可以按用户名或内容过滤。日志写在 `audit.log`，超过 8MB 自动轮转成 `audit.log.1`。

## 几点说明

- 会话过期时页面会提示 3 秒再跳登录页，未同步的改动不会静默丢失。
- 图片在浏览器端就压到最长边 720px 再上传，只收 PNG / JPG / WebP。
- 页面不请求任何外部 CDN 或字体，断网的内网环境也能正常打开。
- 「导出 JSON」「从备份恢复」「恢复出厂数据」影响的是**所有人**看到的那份数据，不是你本地的副本。
- **手机端**：侧栏变抽屉，触控目标放大。但对位表是 5 列 × 11 行的二维表，**手机上不可能不横滚**——硬折成一维卡片流会丢掉"横向对比"这个核心价值。所以保留横滚 + `scroll-snap`，每次滑动干净地停在一个品牌列上。
- 产品图现在是**原图**，不压缩。一张 8MB 的图就是 8MB，图多了记得留意 `uploads/` 的体积。
- 评论库单独放 `reviews/reviews.jsonl`，不进 `db.json`。7 个品牌 6,332 条约 4.2MB，聚合一次 190ms。超过 20 万条再考虑换 SQLite。
